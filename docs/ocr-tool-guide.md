# OCR 文字识别工具使用指南

## 功能概述

**OCR（光学字符识别）工具** 可以从遥感影像中提取文字内容，适用于识别卫星图、航拍图和扫描地图上的文字标注。

- **技术实现**：PaddleOCR v4 引擎
- **支持语言**：中文、英文
- **工具名称**：`ocr_recognize`

---

## 适用场景

### ✅ 适合使用 OCR 的情况

1. **地名标注识别**
   - 卫星图上的城市名、街道名、地标名称
   - 示例："识别这张卫星图上的地名"

2. **建筑物标签提取**
   - 建筑物上的名称标识（商场、车站、学校等）
   - 示例："提取影像中的建筑物名称"

3. **扫描地图文字识别**
   - 纸质地图扫描件的文字注记
   - 图例、标题、说明文字
   - 示例："识别这张扫描地图上的所有文字"

4. **图面注记提取**
   - 遥感图件上的人工标注文字
   - 坐标、方位、备注信息
   - 示例："读取图中的坐标标注"

### ❌ 不适合使用 OCR 的情况

- **纯地物影像**（农田、森林、水体）→ 使用 `segment_landcover`（地物分类）
- **目标检测**（飞机、舰船、车辆）→ 使用 `detect_objects`（目标检测）
- **植被分析**（NDVI、作物健康）→ 使用 `calculate_ndvi`（植被指数）

## 使用示例

### 示例 1：识别卫星图地名

**用户提问：**
```
这张卫星图上有哪些地名？
```

**智能体响应：**
```
调用工具：ocr_recognize(imagery_id="abc123def456")

识别结果：
- 深圳市政府（置信度 0.95）
- 市民中心（置信度 0.92）
- 福田区（置信度 0.88）
```

### 示例 2：扫描地图文字提取

**用户提问：**
```
提取这张扫描地图上的所有文字内容

**智能体响应：**
```

- 地图标题、图例说明
- 行政区划名称
- 坐标标注

## 工具参数说明

### 必需参数

| 参数 | 类型 | 说明 |
|------|------|------|
| `imagery_id` | string | 已上传影像的 ID（12位十六进制） |

### 可选参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `red_band` | integer | 1 | RGB 合成用红光波段号 |
| `green_band` | integer | 2 | RGB 合成用绿光波段号 |
| `blue_band` | integer | 3 | RGB 合成用蓝光波段号 |
| `grayscale` | boolean | false | 灰度模式（扫描件/单波段图建议开启） |
| `max_dimension` | integer | 2048 | 识别前最长边缩放上限（像素，256-8192） |
| `min_confidence` | float | 0.0 | 置信度过滤阈值（0-1，建议 0.7） |

## 最佳实践

### 1. 提高识别准确率

- **扫描地图**：使用 `grayscale=True`
- **多波段影像**：根据实际波段顺序调整 RGB 参数
- **过滤噪声**：设置 `min_confidence=0.7` 过滤低质量结果

### 2. 处理大尺寸影像

- 默认 `max_dimension=2048` 适合大多数场景
- 超大影像（>8192px）会自动缩放，不影响文字识别
- 过小的文字（<10px）可能无法识别

### 3. 常见问题

**Q: 为什么识别不到文字？**
- 检查影像是否真的包含文字（纯地物影像无文字）
- 文字可能过小、模糊或被遮挡
- 尝试降低 `min_confidence` 阈值

**Q: 识别结果中有乱码怎么办？**
- 提高 `min_confidence` 阈值（推荐 0.7-0.9）
- 检查影像质量和文字清晰度

**Q: 如何识别特定区域的文字？**
- OCR 工具会识别整幅影像，无法指定区域
- 建议先裁剪影像再上传，或在结果中筛选坐标范围
